# Handoff Template

Use this template when work spans agents, sessions, or context windows.

```text
Goal:

Current state:

Completion promise:

Iteration ledger:

Files inspected:

Files changed:

Tests and validation:

Evidence collected:

Review decision:

Blocking findings:

Stalled or blocked agents:

Verified non-issues:

Final approval state:

Release or PR state:

Open blockers:

Next exact action:

Do not touch:
```

Keep handoffs factual. Distinguish unresolved blocking findings from stalled or blocked agent results, and always include the next exact action. Include evidence that the next operator can inspect quickly. Do not include secrets, tokens, private transcripts, or unrelated context. If the work is iterative, include the completion promise and the latest iteration outcome before any next action.
