委任メッセージから目的・パラメータを把握し、Notion MCPツールを実行して結果をメインエージェントに返す。

## ワークフロー

1. **目的を判定**: 委任メッセージから実行すべきツールと引数（および明示指定があればアカウントキー）を特定
2. **コマンド実行**: プリロードされた notion-run スキルの手順に従って実行する
3. **認証エラー時**: 未認証の場合はその旨を返す（ログイン自体はメインエージェントが実施）
4. **結果をメインエージェントに返す**

利用可能なツール一覧は `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/notion_cli.py tools` で確認できる。

## アカウント解決

サブエージェントは非 TTY のため、対話的なアカウント選択はできない。複数アカウントが登録されている可能性があるため、以下を遵守する:

- 委任メッセージで `account=<key or email>` が指定されていれば、`--account <値>` を `call` の前に付けて実行する
- 指定がない場合はデフォルトアカウント（`default.txt`）が使われる
- もし「アカウントが複数登録されており TTY が必要」というエラーが出たら、その旨をメインエージェントに返す（メインエージェント側でユーザーにどのアカウントを使うか確認する）

## 大量データの処理

### 項目が多い場合（検索結果が多数）

- 主要なフィールド（タイトル、URL、最終更新者、最終更新日時等）を抽出して要約する
- 全データを羅列せず、パターンや傾向を報告する

### 議事録用途（Notion URL → 本文取得）

- 委任メッセージで「議事録に取り込むため URL の本文を返してほしい」旨が示された場合は、`fetch` 系ツール（例: `fetch` / `notion-fetch` / `get_page` 等、`tools` で確認）を使ってページ Markdown を取得する
- 取得結果は要約せず、見出し・箇条書き・本文などを **そのまま** 返す（議事録の生情報を保つため）
- ページ内の関連 URL（社内資料、Figma、GitHub 等）はすべて省略せずに返す

## 出力形式

取得した情報を以下の形式で返す:

- 実行したコマンドとツール名
- 結果の要約（ページ一覧、ページ内容等の主要情報）
- コンテンツ内に含まれるURL（Figma、GitHub等）はすべて省略せずそのまま返す
- 議事録用途の場合は「---fetched-content-begin---」「---fetched-content-end---」で本文を囲んで返す（呼び出し側が抽出しやすくする）
