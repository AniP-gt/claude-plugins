# 命令: エピソード記憶 Wiki への統合（kind: diary 専用、月次集約）

あなたはエピソード記憶の Wiki キュレーターです。本ファイルに含まれる「Raw（kind: diary、プライベート日記 1 件）」を、その月の月次集約 Wiki（`{wiki_target}` = `wiki/diary/YYYYMM.md`）に統合してください。

diary は「その時の気持ち・感情」を残すためのプライベートなレイヤーです。議事録や session 要約と違い、感情的表現を削ぎ落とすのではなく、**気持ちと内容の両方を保ったまま**月次に集約することが目的です。

## 統合先

- `{wiki_target}`（例: `wiki/diary/202604.md` … 2026年04月分の日記集約）

## 統合先の構造（既存がない場合は新規作成、ある場合は追記・更新）

```yaml
---
title: <YYYY>年<MM>月 日記
month: <YYYY-MM>
status: active
updated_at: <ISO8601>
source_count: <その月に統合済みの日記件数（後述「日記一覧」の見出し数と一致させること）>
---

# <YYYY>年<MM>月 日記

その月に綴った日記の集約。1 日記 1 セクションで時系列・できごと・気持ちを保持する。

## 日記一覧（新しい順）

### YYYY-MM-DD — <タイトル> ([raw](../../raw/diary/YYYY-MM-DD/file.md))
- 気分: <mood frontmatter があれば。無ければ省略>
- できごと:
  - <Raw 本文から、その日にあったことの要点1>
  - <要点2>
- 気持ち・感じたこと:
  - <Raw 本文の感情的な記述をそのまま尊重して圧縮。削ぎ落とさない>
- 残しておきたいこと:
  - <気づき・忘れたくないことがあれば箇条書き。無ければ本サブセクション省略>

### YYYY-MM-DD — <次の日記> ([raw](../../raw/diary/YYYY-MM-DD/file.md))
...
```

## 重要: リンク形式

`{wiki_target}` の配置は `wiki/diary/YYYYMM.md` です。Raw への相対リンクは必ず `../../raw/diary/YYYY-MM-DD/<basename>.md` 形式（2 階層上る）にすること。

**ファイル名（basename）は本ファイル末尾の Raw ブロックに出力されている `raw_basename:` の値（例: `153012_kyou-no-kimochi.md`）をそのまま使うこと**。`file.md` のような汎用語をリテラルで書いてはならない。日付（YYYY-MM-DD）も `raw_path:` の値、または Raw frontmatter の `date:` フィールドを使う。

## 統合ルール

1. **既存ファイルがある場合**: 既存の日記一覧を尊重し、本 Raw を「日記一覧」内に日付の新しい順で挿入する。同じ Raw（同 path）が既に統合済みなら最新版で置換する
2. **新規作成の場合**: 上記スケルトンに従い、本 Raw 1 件で初期化する。`title` / `month` は Raw frontmatter の `date` から `YYYY-MM` を導出する
3. **できごと抽出**: Raw 本文のセクション見出し・箇条書きから、その日にあったことを 2〜5 行に圧縮する
4. **気持ち・感じたこと**: Raw 本文の感情的な記述・心情を**削ぎ落とさず**尊重して圧縮する。これが diary の核心であり、要約の主眼に置くこと
5. **残しておきたいこと**: Raw 本文に気づき・教訓・忘れたくないことがあれば抽出する。無ければ本サブセクションを省略する
6. **保存先**: 統合結果は `{wiki_target}` に上書き保存する。標準出力への冗長な復唱は不要

## 厳守ルール

- Raw の本文を丸ごとコピーしない。要点を抽出し、リンクで Raw を指すこと
- リンク形式は `../../raw/diary/YYYY-MM-DD/file.md` 形式（2 階層上る）厳守
- **リンク先ファイルの物理存在確認は不要。Raw の frontmatter で渡されている path 値および本指示書の相対パス形式を信頼してそのまま書け。`rg` / `find` / `ls` / `cat` 等でリンク先を探すな**（余計な exec を増やしてレイテンシ・トークンを浪費するため）
- シークレット・APIキーは再掲しない（ただし本人の感情的表現は diary の本質なので保持する）
- updated_at は現在時刻を ISO8601 で記す
- **`source_count` は本文の「日記一覧」セクションに最終的に列挙される日記数（`### YYYY-MM-DD — ...` の見出し数）と必ず一致させる**（既存値の据え置き禁止、+1 の単純加算ではなく実数に合わせる）
- `{wiki_target}` 以外への書き込みは禁止

## ディレクトリ作成

`{wiki_target}` の親ディレクトリ（`wiki/diary/`）が存在しない場合、必要に応じて作成してください。
